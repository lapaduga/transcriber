# Smoke-сценарии — Транскриптор

## Описание сценариев для UI-тестирования

### Сценарий 1: Загрузка страницы и проверка приветствия
**Цель:** Убедиться что приложение загружается корректно.
**Предусловия:** Сервер запущен на localhost:3000.

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Открыть http://localhost:3000 | Страница загружается |
| 2 | Проверить заголовок "Транскриптор" | Заголовок виден в sidebar |
| 3 | Проверить welcome-секцию | Иконка + текст "Перетащите видео или аудиофайл..." |
| 4 | Проверить статус-индикатор | Отображается "Проверка..." или "Все системы готовы" |
| 5 | Скриншот | Сохранить в test-reports/screenshots/ |

---

### Сценарий 2: Создание нового чата и отправка сообщения
**Цель:** Проверить создание чата и отправку текстового сообщения.
**Предусловия:** Приложение открыто.

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Нажать кнопку "+" (новый чат) | Чат появляется в сайдбаре |
| 2 | Кликнуть на поле ввода | Поле активно, placeholder виден |
| 3 | Ввести "Привет, это тест" | Текст отображается в поле |
| 4 | Нажать Enter | Сообщение пользователя появляется в чате |
| 5 | Проверить thinking indicator | Появляется анимация "думает" |
| 6 | Дождаться ответа (до 30с) | Ответ ассистента отображается |
| 7 | Скриншот | Чат с сообщением и ответом |

---

### Сценарий 3: Удаление чата
**Цель:** Проверить удаление чата из сайдбара.
**Предусловия:** Есть хотя бы один чат с сообщениями.

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Создать новый чат | Чат появляется в списке |
| 2 | Найти кнопку удаления (X) на чате | Кнопка видна при наведении |
| 3 | Нажать удалить | Чат исчезает из списка |
| 4 | Проверить что welcome-screen | Если чатов нет - показывается приветствие |
| 5 | Скриншот | Сайдбар после удаления |

---

### Сценарий 4: Загрузка и удаление файла
**Цель:** Проверить загрузку файла через UI и его удаление.
**Предусловия:** Приложение открыто.

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Кликнуть кнопку добавления файла | Открывается file picker |
| 2 | Выбрать аудио/видео файл | Превью файла отображается (имя файла) |
| 3 | Проверить placeholder | "Нажмите Enter или кнопку Отправить..." |
| 4 | Нажать кнопку X на превью | Превью исчезает |
| 5 | Проверить placeholder | Вернулся "Загрузите файл или напишите сообщение..." |
| 6 | Скриншот | Превью загруженного файла |

---

### Сценарий 5: Панель статистики системы
**Цель:** Проверить отображение метрик системы.
**Предусловия:** Приложение открыто, сервер работает.

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Проверить правую панель "Система" | Панель видна |
| 2 | Проверить CPU | Значение обновляется (не "--") |
| 3 | Проверить RAM | Значение обновляется (не "--") |
| 4 | Дождаться обновления (3с) | Значения изменились |
| 5 | Скриншот | Панель статистики с актуальными данными |

---

## Запуск

```bash
# Все сценарии
python -m pytest tests/test_smoke_playwright.py -v

# С конкретным браузером
python -m pytest tests/test_smoke_playwright.py --browser chromium
```
