# Profiles — Многопрофильный ассистент

## Назначение

Три профиля для агентного режима работы с кодовой базой проекта transcriber. Каждый профиль — изолированный контекст с чёткими границами: что делать, что не делать, как оформлять результат.

## Профили

| Профиль | Файл | Миссия |
|---------|------|--------|
| Bug Fix | `bug-fix.md` | Найти баг, починить, проверить |
| Research | `research.md` | Исследовать кодовую базу, дать структурированный ответ |
| Code Review | `code-review.md` | Проверить изменения, найти проблемы, дать рекомендации |

## Как использовать

### В Claude Code / Cursor / opencode

Скопируй содержимое нужного профиля в system prompt или добавь как context:

```
/load profiles/bug-fix.md
```

Или вставь в начало промпта:

```
Следуй профилю из profiles/bug-fix.md для этой задачи.
```

### Активация по контексту

| Ключевые слова в задаче | Профиль |
|--------------------------|---------|
| "баг", "ошибка", "не работает", "сломалось", "fix", "bug" | Bug Fix |
| "как работает", "объясни", "исследуй", "найди как", "research" | Research |
| "проверь", "review", "rev", "MR", "PR", "дифф" | Code Review |

```
                    ┌─────────────┐
                    │  ВХОДНАЯ     │
                    │  ЗАДАЧА      │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │ Bug Fix  │ │ Research │ │  Review  │
        │ 6 states │ │ 6 states │ │ 5 states │
        └──────────┘ └──────────┘ └──────────┘
              │            │            │
              ▼            ▼            ▼
        ┌──────────────────────────────────┐
        │          РЕЗУЛЬТАТ              │
        │  (фикс / ответ / отчёт)         │
        └──────────────────────────────────┘
```

## Invariants (общие для всех профилей)

1. **Язык** — русский (как и весь UI проекта)
2. **Архитектура** — two-server (server.py + mcp_server.py)
3. **Состояние** — in-memory only (conversations, jobs, stream_results)
4. **Секреты** — только через .env + os.getenv()
5. **Блокировка** — нет blocking I/O в route handlers
6. **Качество** — ruff check + mypy после каждого изменения
